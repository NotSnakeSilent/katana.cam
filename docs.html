    Item 1.1 - AntiScreenShare

    Katana AntiScreenShare Module offers a solution to automatic screenshare tools to prevent you from being detected when using cheats.

    Currently, we support the following automatic ss tools

    • Paladin - The most competent anti-cheat solutions
    • Avenge - The #1 Screenshare tool on the market
    • Echo - The most effective tool ever seen
    • Safeshare - The longest running screen-share tool
    • Companion - Breakthrough Anti-Cheat systems

    Usage

    1. Download Katana
    2. Select the AntiScreenshareTool (Number 1)
    3. Set it up

    Options

    The options vary according to the version of Katana used, the variations are as follows

  • Free Version: Katana will crash all the ss tools mentioned above, this may seem a bit blatant to the staff, you can't choose which ss tools to crash.
  • VIP Version: Katana will crash the ss tools you select.
  • VIP+ Version: Same as the VIP version, but you can choose between other methods apart from crashing ss tools, there is an explanation exactly below.

    • VIP+ Methods

    The VIP+ version offers more methods to avoid the automatic screenshare tools, which are

  • FakeScan: Katana will replicate the previously selected ss tool gui, which will initiate a "fake" scan, in which the results will never reach the staff.
  • OpenWith Bug: Katana will use a Windows bug to make the screenshare tool executable previously selected unrecognizable.
  • Missing DLL: Katana will cause the screenshare tool executable to not find the necessary dlls to run.
  • NoDiskSpace: Katana will make the screenshare tool executable think that there is no disk space.

    Item 1.6 - DLL Inyector

    The name says all , use this to inject an DLL to any process

    We offer this Injection Methods:

    • Standard: this is the injection technique used in almost all existing injectors. It uses CreateRemoteThread and LoadLibrary and is the most reliable injection technique.
    • LdrLoadDll Stub: this is similar to the standard injection technique, except that it goes one level deeper into LoadLibrary.
    • LdrpLoadDll Stub: this goes even deeper into LdrLoadDll. It can crash or cause errors on operating systems newer than Windows 10, as it can change.
    • Thread hijacking: this is a fairly stable injection method that takes over already running code to inject its DLL and is not used by many injectors.
    • Manual map: this is the most secure injection technique. So secure that even Windows will not even know about the injected DLL. It may not work correctly on operating systems newer than Windows 10, the injector will tell you what to do in that case.

    And this PostInjection Options

    • Clear PE: clears the PE headers at the start of the injected DLL, making it difficult for them to identify that a DLL exists in a specific location.
    • Hide module: hides the DLL from the list of modules of the process, so if they searched in the list of modules of a process, it would not appear.
